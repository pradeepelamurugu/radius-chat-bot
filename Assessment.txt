Build a Simple Chat Experience
What we’re looking for
We’d like you to build a simple chat experience that feels like something real users could use.
Imagine you’re building a lightweight chat feature for a product — something that allows people to
communicate with each other instantly.
This is not just about making it work, but about how you think through the experience, trade-offs,
and overall quality.
The Problem
Users should be able to open a web page and chat with each other in real time.
Messages should appear instantly without needing to refresh the page.
Beyond that, we’re intentionally leaving things open for you to decide.
What the experience should roughly support
At a minimum, users should be able to:
• Send messages
• See messages from others appear live
• Understand who sent which message
Everything else is up to you.
For example:
• Should messages be saved or disappear on refresh?
• Should users have names or identities?
• Should there be one chat or multiple conversations?
There’s no single correct answer — we’re interested in how you think about these decisions.
Make it feel like a real product
We’d like you to go a bit beyond the basics and add at least one feature that improves the
experience.
More important than the feature itself is:
• Why you chose it
• How it improves the experience
What we care about
We’re not expecting a polished production app, but we do care about:
• Clarity – Is it easy to understand and use?
• Thoughtfulness – Did you make intentional decisions?
• Reliability – Does it work smoothly?
• Simplicity – Did you avoid unnecessary complexity?
Final note
There’s intentionally some ambiguity in this assignment. In real product work, requirements are
often not fully defined — we’re interested in how you navigate that.
